Fewer Tokens = More Accurate

A March 2026 paper found that constraining models to brief responses improved accuracy by 26 percentage points. Fewer tokens means fewer opportunities for hallucination. The "Caveman" technique achieves 65% average output reduction across diverse tasks.

This isn't just a cost optimization — it's a reliability technique. Verbose output gives the model more chances to drift off course.

Key insight: Compression is context-dependent. Never compress code generation or user-facing docs. Compress agent-to-developer communication aggressively.

Design Systems → Design Skills → Generative Protocols

Three layers of UI generation have emerged over a decade:

Layer 1 (2015-2023): Static Design Systems — Humans design upfront. Components are fixed. AI has no role.

Layer 2 (2024-2025): Design Skills — AI generates code, but design skills inject taste as correction layers. The AI builds, humans curate quality.

Layer 3 (2025-2026): Generative Protocols — AI outputs structured UI descriptions (A2UI, AG-UI) at runtime. Designers become taste architects encoding principles, not pushing pixels.

Key insight: Each layer doesn't replace the previous — it builds on it. Design systems are still needed; design skills still correct; protocols just add the runtime generation capability.

OKLCH: Perceptually Uniform Color

/* HSL: yellow and blue at same "lightness" look wildly different */
.yellow { color: hsl(60, 100%, 50%); }  /* looks bright */
.blue   { color: hsl(240, 100%, 50%); } /* looks dark */

/* OKLCH: same lightness actually looks the same */
.yellow { color: oklch(0.80 0.18 95); }
.blue   { color: oklch(0.80 0.18 260); }
/* Both genuinely appear same brightness */


For AI-generated UI, OKLCH is essential. When agents pick colors programmatically, perceptual uniformity prevents the "why does this palette look unbalanced?" problem that plagues HSL-based generation.

Every Tech Wave Follows the Same Cycle

From the 1970s microprocessor to 1990s OOP to 2000s Agile to 2020s AI — every major tech disruption follows: Disruption → Chaos → Practitioners keep building → Good practices emerge → New equilibrium.

The "this time is different" illusion repeats too. OOP prompted abandoning structured programming. AI is prompting abandoning tests ("the model will handle it"). The pattern says: craft never becomes obsolete. It becomes more visible.

Key insight: Well-structured code (small functions, DRY, CI/CD) helps agents MORE than it helps humans. Messy codebases that humans navigate via tribal knowledge become blockers when agents try autonomous changes.
