Agent Interoperability — The A2A Protocol

Sources

  • Announcing the Agent2Agent Protocol (Google Developers Blog) — launch announcement, April 9 2025, design principles, protocol details, 50+ partner endorsements

The Problem: Agents Can’t Talk to Each Other

Enterprise AI deployments are fragmented. A Salesforce agent, a ServiceNow agent, and a SAP agent all work independently — they can’t coordinate across systems because they’re built on different frameworks and speak incompatible APIs.

The missing layer: a standard protocol for agents to discover, communicate with, and coordinate other agents — just as HTTP gave web pages a universal linking layer.

Key insight: A2A to multi-agent systems is what HTTP was to websites — the interoperability layer that enables a heterogeneous ecosystem to function as one coherent network.

What A2A Is

Agent2Agent (A2A) is an open protocol (launched April 2025 by Google Cloud, 50+ partners) for AI agents to:

  • Discover each other’s capabilities
  • Communicate tasks and results
  • Coordinate actions across different enterprise platforms
  • Work together regardless of underlying vendor or framework

A2A complements Anthropic’s MCP: MCP gives agents tools and context. A2A gives agents the ability to delegate to and coordinate with other agents. They layer, not compete.

How A2A Works

Two roles:

  • Client agent — formulates and sends tasks
  • Remote agent — receives tasks, acts, returns results

Capability Discovery — Agent Cards

{
  "name": "CandidateSourcingAgent",
  "version": "1.0",
  "capabilities": ["candidate-search", "skill-matching"],
  "authentication": { "schemes": ["oauth2"] },
  "endpoint": "https://recruiting.example.com/a2a"
}

Client agents query Agent Cards to dynamically find the right specialist. No hardcoded routing.

Task Lifecycle

{
  "id": "task-abc123",
  "status": "in_progress",
  "input": { "job_id": "eng-234", "location": "SF" },
  "artifact": null,
  "updates": []
}

Status: created → in_progress → completed | failed. Supports real-time updates for long-running tasks (hours or days with human-in-the-loop).

Collaboration Messages

Agents exchange messages with typed “parts”:

{
  "parts": [
    { "type": "text", "content": "Found 12 matching candidates." },
    { "type": "data", "content_type": "application/json", "data": [...] }
  ]
}

Parts can include text, data, images, video, or A2UI UI components. Content-type negotiation for the right format.

A2A Client — TypeScript + PHP

// TypeScript: minimal A2A client — send a task to a remote agent
// Based on: https://developers.googleblog.com/en/a2a-a-new-era-of-agent-interoperability/

interface A2ATask {
  id: string;
  status: "created" | "in_progress" | "completed" | "failed";
  input: Record<string, unknown>;
  artifact: unknown | null;
  updates: Array<{ message: string; timestamp: string }>;
}

async function sendA2ATask(
  agentEndpoint: string,
  input: Record<string, unknown>,
  authToken: string
): Promise<A2ATask> {
  const response = await fetch(`${agentEndpoint}/tasks`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "Authorization": `Bearer ${authToken}`,
    },
    body: JSON.stringify({ input }),
  });
  if (!response.ok) throw new Error(`A2A error: ${response.status}`);
  return response.json() as Promise<A2ATask>;
}

async function pollUntilComplete(endpoint: string, taskId: string, token: string): Promise<A2ATask> {
  while (true) {
    const res = await fetch(`${endpoint}/tasks/${taskId}`, {
      headers: { "Authorization": `Bearer ${token}` },
    });
    const task = await res.json() as A2ATask;
    if (task.status === "completed" || task.status === "failed") return task;
    await new Promise(r => setTimeout(r, 1000)); // poll every second
  }
}

// Orchestrator delegates to specialist agents across vendor boundaries
async function orchestrateHiring(jobId: string, token: string): Promise<void> {
  const sourcing = "https://recruiting.acme.com/a2a";
  const scoring  = "https://scoring.hr-tools.com/a2a";
  const calendar = "https://calendar.enterprise.com/a2a";

  const candidateTask = await sendA2ATask(sourcing, { job_id: jobId, location: "SF" }, token);
  const candidates    = await pollUntilComplete(sourcing, candidateTask.id, token);

  const scoredTask = await sendA2ATask(scoring, { candidates: candidates.artifact }, token);
  const scored     = await pollUntilComplete(scoring, scoredTask.id, token);

  await sendA2ATask(calendar, { top_candidates: scored.artifact, job_id: jobId }, token);
  // Each specialist from a different vendor — A2A handles the protocol
}


<?php
// PHP: send an A2A task and poll for completion
function sendA2ATask(string $endpoint, array $input, string $token): array {
    $ch = curl_init("$endpoint/tasks");
    curl_setopt_array($ch, [
        CURLOPT_POST        => true,
        CURLOPT_POSTFIELDS  => json_encode(['input' => $input]),
        CURLOPT_HTTPHEADER  => ["Content-Type: application/json", "Authorization: Bearer $token"],
        CURLOPT_RETURNTRANSFER => true,
    ]);
    return json_decode(curl_exec($ch), true);
}

function pollA2ATask(string $endpoint, string $taskId, string $token): array {
    do {
        sleep(1);
        $ch = curl_init("$endpoint/tasks/$taskId");
        curl_setopt_array($ch, [CURLOPT_HTTPHEADER => ["Authorization: Bearer $token"], CURLOPT_RETURNTRANSFER => true]);
        $task = json_decode(curl_exec($ch), true);
    } while (!in_array($task['status'] ?? '', ['completed', 'failed'], true));
    return $task;
}

// Delegate work across vendors — no custom glue code per integration
$candidates = sendA2ATask('https://recruiting.acme.com/a2a', ['job_id' => 'eng-234'], $token);
$done       = pollA2ATask('https://recruiting.acme.com/a2a', $candidates['id'], $token);
// $done['artifact'] → structured candidate list, ready to pass to next agent
