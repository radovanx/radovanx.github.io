Building Reliable AI Systems — Patterns That Actually Work

Synthesis across 7 sources spanning AI agents, security, and knowledge management.

AI agents are simultaneously capable of superhuman feats and fundamentally unreliable. This page connects insights from across the wiki to identify which architectural patterns actually make AI systems reliable.

The Paradox

Anthropic’s Glasswing ([[ai-security/glasswing-project]]) deploys AI to find security vulnerabilities that escaped 27 years of human review and 5 million automated tests. Yet research shows ([[ai-agents/llm-reliability-limits]]) that LLMs fail at multi-step reasoning — even 95% per-step accuracy compounds to 7.7% over 50 steps.

Resolution: AI excels at pattern matching at scale (scanning millions of code paths for vulnerability patterns) but fails at sequential reasoning (chaining 50 dependent logic steps). The architecture must match the task to the capability.

The Three Patterns

Every reliable AI system in our wiki uses one or more of these:

1. Knowledge Compilation (not RAG)

Instead of asking an LLM to derive answers from raw data each time (unreliable), pre-compile knowledge into structured artifacts where errors are visible and correctable.

  • The [[knowledge-management/llm-wiki-pattern|LLM wiki pattern]] pre-compiles knowledge into cross-referenced pages
  • Glasswing pre-compiles vulnerability signatures into scanning patterns
  • This is why wikis beat chatbots for knowledge work — synthesis is durable, not ephemeral

2. Tool Search (not Tool Loading)

Instead of loading all capabilities upfront (context overflow), discover tools dynamically based on the current task.

  • [[ai-agents/tool-search|Claude’s tool search]] loads 3-5 relevant tools per task instead of 50+
  • This reduces hallucination risk — fewer irrelevant tools means fewer wrong tool calls
  • Scales to 10,000+ tools without degrading accuracy

3. Persistent Memory (not Stateless)

Instead of re-deriving context every session (expensive, lossy), accumulate knowledge across sessions.

  • [[ai-agents/persistent-agent-memory|Persistent agent memory]] tracks history across interactions
  • File-based memory (like CLAUDE.md) gives agents stable context
  • Vector stores enable semantic recall of past interactions

The Meta-Pattern

All three patterns share the same insight: move work out of the unreliable real-time LLM call and into reliable pre-computed infrastructure.

Implications for This Wiki

This project embodies all three patterns:

  • Knowledge compilation: Raw sources → synthesized wiki pages with cross-references
  • Future: tool search: Load only relevant wiki pages for each query (see PRD Phase 3)
  • Persistent memory: The wiki itself IS the memory — each session builds on previous work

The wiki pattern turns the LLM reliability problem from “can the LLM reason correctly right now?” to “has the LLM compiled correct knowledge over time?” — a much easier problem because errors compound into visibility rather than compounding into failure.
