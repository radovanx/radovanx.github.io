Tool Search — Scaling Agent Tool Discovery

As AI agents gain access to more tools, a fundamental scaling problem emerges: loading all tool definitions into the context window becomes impractical. Claude’s Agent SDK solves this with tool search — dynamic, on-demand tool discovery.

The Problem

  • 50 tools consume 10-20K tokens of context just for definitions
  • Tool selection accuracy degrades beyond 30-50 loaded tools
  • Large tool catalogs (hundreds or thousands) are unusable without search

How It Works

Tool definitions are withheld from the context window. The agent receives a summary of available capabilities and searches for relevant tools when a task requires one. The 3-5 most relevant tools are loaded into context and remain available for subsequent turns.

This adds one extra round-trip on first discovery but saves tokens on every subsequent turn.

Configuration

The ENABLE_TOOL_SEARCH environment variable controls behavior:

ValueBehavior
true (default)Always on — tool definitions never loaded upfront
autoActivates when tool definitions exceed 10% of context
auto:NCustom threshold (e.g., auto:5 for 5%)
falseOff — all tools loaded into context every turn

Limits

  • Maximum 10,000 tools in the catalog
  • Returns 3-5 most relevant tools per search
  • Haiku models do not support tool search

Optimization

Tool names and descriptions drive search quality. search_slack_messages surfaces better than query_slack. Adding a system prompt listing available tool categories helps the agent know what to search for.
