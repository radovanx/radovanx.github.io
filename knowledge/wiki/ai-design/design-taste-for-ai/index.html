Design Taste for AI

AI can generate UI code instantly, but it has no taste by default. Left uncorrected, LLMs produce generic, flat, “slop” interfaces — the visual equivalent of filler text. The emerging solution: design skills — injected knowledge that overrides LLM biases with specific, opinionated design rules.

The Problem

LLMs default to:

  • Inter/Roboto fonts (safe, boring)
  • Heavy drop shadows (generic depth)
  • Symmetric layouts (no tension)
  • No motion (static, lifeless)
  • Placeholder content (“Lorem ipsum”, “John Doe”)
  • Rounded pill shapes everywhere
  • No optical alignment (mathematically centered but visually wrong)

The result looks “AI-generated” — technically correct but soulless.

The Solution: Design Skills

Design skills are markdown files that inject design knowledge into AI coding agents. They work as correction layers:

  • [[make-interfaces-feel-better]] — 11 specific UX details (text wrapping, concentric radii, animation interruption, optical alignment, shadows over borders)
  • Taste Skill (Leonxlnx) — a full design system with variance/motion/density controls, anti-generic rules, and strict component architecture
  • Minimalist UI — editorial style inspired by Notion/Linear
  • High-End Visual Design — agency-level polish with premium fonts, spacing, depth

Key Principles

Optical > Mathematical. Visually centered beats geometrically centered. The eye doesn’t care about math.

Shadows > Borders. Elevation through subtle box-shadow creates depth. Borders flatten everything.

Motion is meaning. Enter animations, exit animations, staggered reveals — they communicate state changes and hierarchy. But motion must be invisible: present, never distracting.

Concentric radii. Nested elements need proportional border-radius (outer radius = inner radius + padding). This is the detail that separates polished from amateur.

Tabular numbers for dynamic content. Digits that shift layout when values change look broken. font-variant-numeric: tabular-nums fixes it.

The Bigger Picture

This is a new design pattern: design on demand. Instead of building design systems upfront, you encode design taste into AI skills and generate interfaces that are already polished. The skill IS the design system — portable, versionable, composable.
