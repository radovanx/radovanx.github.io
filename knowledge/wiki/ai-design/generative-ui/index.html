Generative UI — Design on Demand

The idea that UI doesn’t need to be designed upfront — it can be generated at runtime by AI agents, shaped by context, content, and user intent.

Three Layers of the Shift

Layer 1: Static design systems (2015–2023) Design tokens, component libraries, Figma. Humans design everything. AI generates code from designs.

Layer 2: Design skills for AI (2024–2025) Taste skills, prompt engineering for UI. AI generates UI code, but needs injected design knowledge to avoid generic output. See [[design-taste-for-ai]].

Layer 3: Generative UI protocols (2025–2026) [[a2ui-protocol]] and AG-UI. AI agents output structured UI descriptions. The interface materializes on demand — forms, dashboards, cards, charts — based on what the agent needs to communicate.

The Convergence

The three layers aren’t replacements — they stack:

  • A2UI gives agents the vocabulary to describe UI
  • Design skills give agents the taste to make it beautiful
  • Design systems give clients the components to render it natively

What This Means

  • No more designing screens for every state — the agent generates what’s needed
  • Design becomes about rules and constraints (skills), not individual layouts
  • UI testing shifts from visual regression to schema validation
  • Designers become taste architects — encoding principles, not pushing pixels

This Wiki as an Example

This knowledge wiki itself follows the pattern: content is ingested automatically, the LLM generates structured pages on demand, and the newsletter is a designed artifact produced from raw data. The wiki IS generative UI — just for knowledge instead of apps.
