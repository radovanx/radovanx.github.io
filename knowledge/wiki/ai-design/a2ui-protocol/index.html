A2UI Protocol — Agent-to-User Interface

A2UI is an open-source specification by Google that standardizes how AI agents generate user interfaces on demand. Instead of text-only responses, agents output structured JSON that describes UI components — the client renders them natively.

How It Works

  1. Agent decides it needs to show a form, chart, card, or layout
  2. Agent outputs JSON describing the UI structure declaratively
  3. Client renders using its own native components (React, SwiftUI, Flutter, etc.)

The separation is key: the agent defines WHAT to show, the client decides HOW to render it. This gives you security, brand consistency, and cross-platform portability.

The 2026 Protocol Stack

A2UI sits at the top of a layered protocol stack for AI agents:

LayerProtocolPurpose
CoordinationA2A (Agent-to-Agent)Agents talking to agents
ToolsMCP (Model Context Protocol)Agents using tools and data
RuntimeAG-UIAgent-to-frontend streaming
InterfaceA2UIAgent generating UI components

Why It Matters

Before A2UI: AI agents return text or markdown. If you want rich UI, you build it yourself.

After A2UI: AI agents can request specific UI components — forms, data tables, charts, interactive cards — on the fly. The UI is generated at runtime, not designed upfront.

This is the shift from static design systems to generative UI — interfaces that materialize on demand based on context.

Connection to Design Skills
